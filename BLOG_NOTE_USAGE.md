# 日常碎碎念功能使用指南

## 概述

这是一个用于记录日常碎碎念、随笔、想法的功能模块。
它基于现有的博客系统（blog_article表），但专门优化用于快速、碎片化的内容记录。

## 特点

- 基于现有blog_article表，不创建新表
- 添加了心情、位置、天气等碎碎念特有字段
- 支持公开/私有控制
- 支持点赞、评论
- 支持按心情、标签搜索
- 默认直接发布（无需审核）

## API接口

### 创建碎碎念
```
POST /api/blog/note
{
  "content": "今天天气真好，适合散步",
  "mood": "happy",
  "tags": ["天气好", "散步"],
  "location": "公园",
  "weather": "晴朗",
  "isPublic": true
}
```

### 更新碎碎念
```
PUT /api/blog/note
{
  "id": 123,
  "content": "今天天气真好，适合散步，而且遇到了老朋友",
  "mood": "happy",
  "tags": ["天气好", "散步", "老朋友"],
  "location": "公园",
  "weather": "晴朗",
  "isPublic": true
}
```

### 获取我的碎碎念
```
GET /api/blog/note/my?pageNum=1&pageSize=20
```

### 获取公开碎碎念
```
GET /api/blog/note/public?pageNum=1&pageSize=20&mood=happy&tag=散步
```

### 获取单条碎碎念
```
GET /api/blog/note/123
```

### 删除碎碎念
```
DELETE /api/blog/note/123
```

### 获取碎碎念数量
```
GET /api/blog/note/count
GET /api/blog/note/count/mood/happy
```

## 数据库字段

在blog_article表中新增了以下字段：
- mood VARCHAR(50) - 心情
- location VARCHAR(200) - 位置
- weather VARCHAR(50) - 天气

## 使用建议

1. **内容长度**：碎碎念建议保持在1-300字之间，适合快速记录
2. **心情值**：建议使用标准值如 happy, sad, excited, calm, anxious, tired, energetic 等
3. **标签**：用逗号分隔的字符串，如 "天气好,散步,心情好"
4. **位置**：可以是具体地点如 "公园", "咖啡馆", "办公室" 等
5. **天气**：可以是 "晴朗", "多云", "雨天", "雪天" 等
6. **公开/私有**：isPublic=true 表示公开，false 表示仅自己可见
7. **默认状态**：碎碎念默认直接发布为 PUBLISHED 状态，无需审核流程

## 前端建议

1. **快速输入框**：提供一个始终可见的快速输入框，像微信的"在看看"功能
2. **心情选择器**: 提供常用心情的快速选择按钮
3. **位置自动填充**: 可选地使用地理位置服务自动填充位置
4. **天气自动获取**: 可选地根据位置和时间自动获取天气
5. **标签推荐**: 基于历史使用推荐常用标签
6. **时间线展示**: 按时间倒序展示碎碎念，类似朋友圈或时间线
7. **心情统计**: 提供心情分布的统计图表
8. **位置地图**: 可以在地图上显示碎碎念的位置分布
